# Data Collection Platform

## Description

Data Collection Platform 的目标是**收集真实人类在移动设备上的交互数据**，从而基于经验数据学习和校准更加真实的算子参数（operator parameters），而不是仅依赖理论假设或文献中的平均值。

### 核心动机

以 **Fitts’ Law** 为例：  
如果我们希望获得更符合真实用户行为的参数，就需要在真实手机环境中采集用户完成指向、点击、滑动等操作时的行为数据。这要求：

- 用户在**真实设备**上完成指定交互任务；
- 系统在后台**精确记录交互过程中的关键指标**；
- 数据可用于后续的统计建模和参数拟合。

### 平台需要支持的实验形式

平台应支持构建一系列**可控的人机交互实验任务**，例如：

- 屏幕上呈现一个明确的任务提示（如 *“请从当前位置滑动到目标区域”*）；
- 提供与任务匹配的具体交互场景（如不同目标大小、距离、方向的 swipe / tap 任务）；
- 用户按照指示完成操作。

在用户完成任务的过程中，系统需要自动记录包括但不限于：

- 起始位置与目标位置；
- 目标大小、距离、方向等任务参数；
- 操作类型（tap / swipe / drag 等）；
- 操作时长、轨迹、速度、误差等行为指标；
- 是否成功完成任务。

### 多实验与用户层级支持

为了获得稳定、可分析的数据，平台还需要支持：

- **单个用户完成多个实验任务**（例如不同参数组合的 Fitts’ Law 任务）；
- 对实验进行结构化组织（session / trial / block）；
- 区分不同用户的数据，并支持跨用户的数据汇总与对比分析。

### 总体定位

综上，Data Collection Platform 的作用可以概括为：

> 一个用于设计、部署和运行移动端人机交互实验的平台，  
> 通过采集真实用户在受控任务下的交互行为数据，  
> 为后续的人机交互建模（如 KLM / CPM-GOMS / Fitts’ Law 等）提供高质量的经验数据支撑。

## 你需要完成的工作

1. **回顾并理解现有的 model-based evaluation 如何构建预测公式与拟合参数**  
   重点关注经典工作中：  
   - 如何定义任务变量（自变量）、性能指标（因变量）  
   - 如何建立公式（例如线性/对数形式、分段形式等）  
   - 如何做参数拟合（回归方法、数据清洗、异常值处理、评价指标）  
   参考阅读材料包括但不限于：  
   - *Fitts' law as a research and design tool in human-computer interaction*  
   - *The keystroke-level model for user performance time with interactive systems*

2. **调研是否存在可复用的用户实验平台工具，并评估其对“手机交互实验”的适配性**  
   已知有一些实验平台/工具可用于快速搭建用户研究，包括但不限于：  
   - [PsychoPy](https://www.psychopy.org/)  
   - [ReVISit](https://revisit.dev/blog/2025/01/20/release-2.0/)  

   你需要完成：  
   - 调研还有哪些常用工具，这些工具是否能**直接用于手机场景**（移动端、触控输入、屏幕尺寸适配、数据采集能力等）  
   - 如果不能直接用，判断是否能以较小代价**改造/扩展**满足需求  
   - 若存在“可直接用”或“非常接近”的方案，优先选择复用或基于其修改  
   - 将调研过程与结论记录下来，用于后续论证（justify）为什么需要自建或改造实验平台（包含对比维度与取舍理由）

3. **若确认无法有效复用现有方案，则先输出设计稿，再进入开发**  
   - 基于当前需求整理功能与数据采集清单，形成一版设计稿（包含实验流程、界面结构、数据记录点、任务配置方式等）  
   - 在与团队讨论并确认设计满足需求后，再开始具体开发工作
